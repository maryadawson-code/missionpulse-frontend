MissionPulse API

The MissionPulse API provides programmatic access to your proposal management data. Built for federal contractors, our API enables seamless integration with your existing workflows while maintaining CMMC 2.0 compliance.

Base URL

https://missionpulse-api.onrender.com/api/v1

Secure by Default

TLS 1.3 encryption, JWT auth, RBAC enforcement

High Performance

Sub-100ms response times, edge caching

CMMC Compliant

Full audit logging, CUI protection

Authentication

MissionPulse uses JWT (JSON Web Tokens) for API authentication. Include your token in the Authorization header of all requests.

Request Header

Authorization: Bearer YOUR_JWT_TOKEN

Obtaining a Token

curl -X POST https://missionpulse-api.onrender.com/api/v1/auth/token \
  -H "Content-Type: application/json" \
  -d '{"email": "user@company.com", "password": "your_password"}'
const response = await fetch('https://missionpulse-api.onrender.com/api/v1/auth/token', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    email: 'user@company.com',
    password: 'your_password'
  })
});
const { token } = await response.json();
import requests

response = requests.post(
    'https://missionpulse-api.onrender.com/api/v1/auth/token',
    json={'email': 'user@company.com', 'password': 'your_password'}
)
token = response.json()['token']

Rate Limits

API requests are rate limited to ensure fair usage and platform stability.

Tier Requests/Minute Requests/Day
Standard 60 10,000
Pro 120 50,000
Enterprise 300 Unlimited
Rate limit headers are included in all responses: X-RateLimit-Remaining, X-RateLimit-Reset

Error Handling

The API uses standard HTTP status codes and returns detailed error messages in JSON format.

Status Description
200 Success
400 Bad Request - Invalid parameters
401 Unauthorized - Invalid or missing token
403 Forbidden - Insufficient permissions (RBAC)
404 Not Found - Resource doesn't exist
429 Rate Limited - Too many requests
500 Server Error - Contact support

Error Response Format

{
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Invalid or expired authentication token",
    "details": {
      "field": "Authorization",
      "hint": "Ensure token is prefixed with 'Bearer '"
    }
  },
  "request_id": "req_abc123xyz"
}
GET

/opportunities

Retrieve opportunities from your pipeline. Supports filtering, sorting, and pagination.

Query Parameters

status string Filter by status: pipeline, pursuing, submitted, won, lost
agency string Filter by agency code (DHA, VA, CMS, etc.)
min_value number Minimum contract value
limit number Results per page (default: 20, max: 100)
offset number Pagination offset

Example Request

curl -X GET "https://missionpulse-api.onrender.com/api/v1/opportunities?status=pursuing&limit=10" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"

Webhooks

Subscribe to real-time events from MissionPulse. Webhooks are sent as POST requests to your configured endpoint with HMAC-SHA256 signatures for verification.

Available Events

opportunity.created New opportunity added
opportunity.status_changed Pipeline stage updated
proposal.submitted Proposal marked as submitted
deadline.approaching 72h before due date

Webhook Payload

{
  "event": "opportunity.status_changed",
  "timestamp": "2026-02-01T10:30:00Z",
  "data": {
    "opportunity_id": "opp_abc123",
    "previous_status": "pipeline",
    "new_status": "pursuing",
    "changed_by": "user_xyz789"
  },
  "signature": "sha256=abc123..."
}
